First Component

First is a component. There is a route defined for it in app.routing.ts.

When a user clicks on the "First" link the component is output to the router-outlet in the AppComponent.

When a user clicks on the "First" link the application requests navigation to the path /first, the router activates an instance of FirstComponent, displays its view, and updates the browser's address location and history with the URL for that path.

I don't know yet if this makes it a child or a sibling of AppComponent. It has been imported to app.component.ts.

From further reading, it looks like the FirstComponent is considered a sibling element to the RouterOutlet that is in the host component's template, AppComponent in this case.
